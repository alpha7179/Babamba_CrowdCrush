## 1. 프로젝트 개요

CrowdCrush VR은 콘서트, 축제, 스포츠 경기, 출퇴근 시간의 지하철 등 **고밀집 환경에서 발생할 수 있는 인파 사고**를 가상현실로 재현합니다. 2022년 이태원 참사를 배경으로 출발했으며, 기존 이론·영상 중심 교육의 한계를 넘어 **실제와 유사한 압박감과 패닉을 체감하고 극복하는 훈련**에 초점을 맞춥니다.

Meta Quest 2/3 헤드셋과 bHaptics TactSuit X40 햅틱 슈트를 함께 활용해 **시각·청각·촉각의 다중감각 피드백**을 제공하며, 단순한 영상 시청만으로는 전달할 수 없는 밀집 압박의 위험성과 긴박감을 사용자 신체에 직접 전달합니다.

### 1.1 핵심 가치

| 항목 | 내용 |
|---|---|
| 몰입형 위기 체험 | 인구 밀집도에 따른 압박감, 파도 현상, 떠밀림 등 실제 사고 현장과 유사한 경험 제공 |
| 제스처 기반 상호작용 | 컨트롤러와 핸드 트래킹을 통해 호흡 공간 확보, 자세 보호 등 생존 행동을 직접 수행 |
| 다중감각 햅틱 피드백 | 인파 압력, 충돌, 심박을 진동으로 전달해 현장감과 긴장감을 극대화 |
| 통제된 심리적 압박 | 터널 비전과 사운드 디자인으로 패닉 상태를 안전하게 경험·극복 |

### 1.2 기대 효과

- 막연한 불안감을 구체적인 대처 행동으로 전환
- 안전한 가상 공간에서의 반복 훈련을 통한 침착한 대응 능력 배양
- 학생, 시민, 안전관리 담당자 등 다양한 계층에 적용 가능한 체감형 안전 교육

## 2. 기획 배경

### 2.1 문제 인식

**대규모 행사 증가와 안전 의식 부족**
콘서트, 축제, 스포츠 경기 등 대규모 인파 행사가 꾸준히 늘어나는 반면, 2022년 이태원 참사에서 드러났듯 개인의 안전 의식과 대처 능력은 여전히 부족한 상황입니다.

**기존 교육 방식의 한계**
이론과 영상 중심의 2D 교육은 실제 상황의 압박감과 혼란을 전달하기 어렵습니다. 그 결과 알고 있는 지식이 위기 순간의 행동으로 이어지지 못하는 **체화 부족** 문제가 반복됩니다.

### 2.2 접근 방향

VR의 높은 몰입도를 활용해 사용자가 *"알고 있는 것"* 을 *"실제로 행동할 수 있는 것"* 으로 전환하도록 돕는 것이 본 프로젝트의 핵심 목표입니다.

## 3. 교육 대상과 목표

### 3.1 대상

- **주요 대상**: 축제, 콘서트, 경기 등 대규모 행사에 자주 참여하는 10~30대 청년층
- **확장 대상**: 학교 안전 교육 프로그램, 행사·시설 안전관리 담당자, 안전 교육에 관심 있는 일반 시민

### 3.2 목표

| 목표 | 설명 |
|---|---|
| 체감형 교육 | 밀집 상황의 위험성을 신체로 체감시키고, 막연한 불안감을 구체적 대처 행동으로 전환 |
| 생존 능력 배양 | 패닉을 유발하는 압박 환경에서 올바른 행동 절차를 반복 훈련해 본능적 대응으로 정착 |
| 위험 인지 훈련 | 떠밀림 등 인파 사고의 전조 증상을 식별하고 비상구를 미리 파악하는 예방 습관 형성 |

## 4. 콘텐츠 구성

### 4.1 씬 디자인

실사풍의 저녁 도시 골목과 지하철 역을 배경으로, 축제 직후와 출퇴근 시간의 혼잡함을 사실적으로 재현합니다.

| 씬 | 설명 |
|---|---|
| 거리 인파 사고 | 축제 직후 좁은 도시 골목, 좌우 상가, 야간 조명, 인파 흐름 재현 |
| 지하철 인파 사고 | 출퇴근 시간 객차 내부와 환승 통로, 계단·개찰구 혼잡 재현 |

### 4.2 위험 감지·행동 유도 UI

현재 위험 수준과 다음에 취해야 할 행동을 직관적으로 시각화합니다.

**플레이어 압력계 (1~6단계)**

| 단계 | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|
| 상태 | 안전 | 경고 | 압박 | 위험 | 마비 | 치명 |

- 1~3단계: 안전부터 압박 초기까지의 구간
- 4~6단계: 위험·마비·치명 구간으로 시청각 피드백 강도가 비선형적으로 증가

**진행 방향 유도로**
바닥에 발광 화살표를 표시해 미션 진행 방향과 안전 경로를 안내합니다.

**자세 유도 및 행동 유지 바**
ABC 자세, 기둥·벽·난간 잡기 등 행동 수행 중 표시되며, 올바른 자세를 3초 유지하면 100% 충전되어 행동 성공으로 판정합니다.

### 4.3 NPC 시스템

다양한 역할의 NPC가 시나리오의 사실감과 인지적 압박을 형성합니다.

| NPC | 동작 |
|---|---|
| 다이내믹 군중 | 밀집 구역에서 서성이거나 웅성거리는 평상시 모션, 패닉 상황에서 소리치는 모션 |
| 피해 시민 | 인파 밀림이나 충돌로 바닥에 쓰러진 상태 |
| 경찰 | 경적을 활용해 시민을 안전 구역으로 유도·통제 |
| 구조대 | 쓰러진 환자에게 CPR과 가슴 압박, 상태 체크 수행 |
| 배회형 시민 | Waypoint 기반 이동으로 외곽 지역에서 자연스럽게 보행 |

### 4.4 사운드 디자인

4개 카테고리, 20여 종의 사운드 큐로 청각적 몰입을 구성합니다.

| 카테고리 | 주요 사운드 |
|---|---|
| UI · Interaction | Button Click/Pop, Success/Level Up, Notification, Process Failed, TTS Voice, Gesture Detect |
| Vitality · States | Heartbeat, Out of Breath, Ear Ringing, Energy Power Up, Pain Groans, Suffering Moans |
| Spatial Ambience | Subway Ambience, Metro Station Walk, Escalator Steady, Subway Squeal, Doors Open, Train Arrival |
| Crowd · Incident | Crowd Talking, Crowd Panic/Scream, Heavy Footsteps, Police Whistle, Ambulance Siren, Phone Calling |

## 5. 시나리오 플로우

콘텐츠는 **인지 → 대처 → 탈출·구호**의 3단계로 구성되며, 사용자는 순차적인 미션을 통해 위기 상황 대처법을 학습합니다.

### 5.1 시나리오 ① — 거리 인파 사고

| Step | 내용 |
|---|---|
| 1 | 체험 장소 선택, 콘텐츠 소개·조작법·대처 행동 요령 확인 |
| 2 | 체험 시 주의사항 안내, 군중 압박 재현에 따른 불편감과 Y 버튼 중단 기능 사전 고지 |
| 3 | 사람이 많은 곳을 피해 가장자리로 이동 |
| 4 | ABC 자세 취하기, 팔로 가슴 공간 확보 |
| 5 | 기둥을 잡아 중심 유지 |
| 6 | 벽을 짚고 호흡 공간 확보 |
| 7 | 경찰·구조대 지시에 따라 안전 구역으로 이동 |
| 8 | 수행 행동 요약과 ★★★ 평가 피드백 제공 |

### 5.2 시나리오 ② — 지하철 인파 사고

| Step | 내용 |
|---|---|
| 1 | 압력 상승 시 가방을 활용한 ABC 자세로 가슴 공간 확보 |
| 2 | 밀집 공간에서 물건을 떨어뜨렸을 때 대처법을 퀴즈 형식으로 학습 |
| 3 | 흐름에 거스르지 않고 동일한 방향으로 이동 |
| 4 | 기둥을 잡아 중심 유지 |
| 5 | 계단 난간을 잡아 넘어짐 방지 |
| 6 | 넘어진 상태에서도 태아 자세로 가슴 보호 |
| 7 | 역무원 지시에 따라 개찰구 대신 비상문으로 이동 |
| 8 | 119 또는 112 신고를 통해 위급 사항 전달 |

## 7. 핵심 인터랙션

### 7.1 상호작용

| 유형 | 구현 |
|---|---|
| Direct Interaction | `XRDirectInteractor` 기반으로 기둥·벽 등 구조물을 직접 잡는 동작, 신체 소유감 강화 |
| Ray Interaction | `XRRayInteractor` 기반 광선으로 UI 패널 조작, 훈련 외 조작 부담 최소화 |
| Gesture Interaction | `GestureManager.cs` 로 팔을 교차해 가슴 공간을 확보하는 ABC 자세 수행 |

### 7.2 이동 방식

| 유형 | 구현 |
|---|---|
| Continuous Move | `Continuous Move Provider`. 텔레포트로 인한 공간 인지 단절을 피하고 실제 재난 현장을 걸어 탈출하는 감각 유지 |
| Continuous Turn | `Continuous Turn Provider`. 실제 몸을 돌리기 어려운 공간에서도 방향 전환 지원 |
| Discrete Adjustment | 멀미에 민감한 사용자를 위한 간헐적 이동 옵션 |

### 7.3 ABC 자세 판정 흐름

```
[Meta Quest 컨트롤러]   ┐
                        ├─► [Cross Trigger] ─► [GestureManager 자세 판정] ─► [StepManager 미션 완료 판정]
[XR 핸드 트래킹]        ┘
```

**판정 로직**
좌·우 가슴에 위치한 두 트리거 존에 양손이 교차하여 동시에 들어왔는지 검사합니다.

**자세 피드백**
1. 트리거 존 진입 시 클릭 효과음과 강한 1회 햅틱 재생
2. 올바른 자세 유지 시 컨트롤러에 약한 연속 햅틱
3. 사용자가 촉각만으로도 현재 상태를 확인할 수 있도록 설계

## 8. 다중감각 피드백 설계

압박 단계(1~6)라는 단일 추상 변수에 대해 시각, 청각, 촉각 채널이 동시 발화되도록 구성했습니다. 단일 감각만으로는 만들어낼 수 없는 정보 전달 강도와 몰입을 의도한 설계입니다.

### 8.1 시각 피드백

압박이 높아질수록 시야가 붉게 좁아지는 비네팅 효과가 게이지·문구와 함께 적용됩니다. 비네팅은 3단계부터 급격히 좁아지도록 곡선을 조정했습니다.

### 8.2 청각 피드백

| 사운드 | 동작 |
|---|---|
| 심박동 | 압력 단계에 비례해 볼륨 증가 |
| 호흡 | 3단계부터 급격히 볼륨 증가 |
| 이명 | 4단계부터 급격히 볼륨 증가 |

군중음과 음성 안내(TTS)가 함께 재생되어 압박도와 위험 상황을 청각으로 전달합니다.

### 8.3 촉각 피드백

| 장비 | 역할 |
|---|---|
| 컨트롤러 햅틱 | 압박 수치 변화를 실시간 반영 |
| bHaptics 슈트 | 특정 방향에서 발생하는 충돌·압박을 신체 전면으로 전달 |

## 9. 이론적 배경

본 프로젝트는 VR 몰입 이론을 기반으로 설계되었습니다.

### 9.1 인지적 경험

| 구분 | 적용 이론 | 프로젝트 적용 |
|---|---|---|
| Place Illusion | Environment / Audio | 사실적인 도시 에셋과 라이팅, 군중 환경음으로 청각적 현장감 구현 |
| Plausibility Illusion | Crowd Animation / Flow | 상황별 군중 애니메이션과 사용자 행동에 따라 변화하는 시나리오 |
| Embodiment | Visual / Locomotion / Interaction | 팔까지 표현된 시각적 연속성, 연속 이동·회전, 직접·제스처 인터랙션을 통한 행동 주체감 |

### 9.2 UI / UX 설계

| 구분 | 적용 이론 | 프로젝트 적용 |
|---|---|---|
| UI | World Space UI | `IntroUIManager.cs` 로 인트로·메뉴를 3D 공간에 배치, 가상 세계와의 시각적 연결 유지 |
| UI | HUD UI | `IngameUIManager.cs` 로 상단 게이지, 중앙 미션, 하단 진행바를 시선 흐름에 맞춰 배치 |
| UX | 다중감각 피드백 | 비네팅, 군중 비명·환경음, 컨트롤러 진동을 위기 상황에 동시 발화 |

### 9.3 멀미 저감

| 이론 | 적용 |
|---|---|
| Vignette | 사용자의 이동 속도와 회전량에 따라 주변부 시야를 즉각적으로 차단 |

## 10. 최적화

다수의 NPC를 동시에 운영하면서도 VR이 요구하는 안정적인 프레임을 확보하기 위한 최적화를 적용했습니다.

### 10.1 NPC 최적화

**적용 기법**
- 거리 기반 렌더링 컬링으로 카메라 거리에 따라 렌더러 비활성화
- 거리 기반 애니메이션 컬링으로 일정 거리 밖 NPC의 애니메이션 중단
- LOD 적용으로 원거리 NPC 폴리곤 자동 감소
- Mesh Baker로 Body, Belt, Shoes, Hair 등 개별 파츠를 병합해 Skinned Mesh 수를 4배 이상 감소

**성능 향상**
- Skinned Mesh Renderer 감소에 따른 CPU 부하 절감
- 400명 NPC 동시 운용 시 40~50 FPS 확보

### 10.2 씬 최적화

**적용 기법**
- GPU Instancing
- 텍스처 크기 최적화
- Cast Shadow 선택적 비활성화와 라이트 베이킹
- Occlusion Culling

**성능 향상**
- Batches 227, Visible Skinned Meshes 46 (에디터 기준)
- PC 연결 VR 환경에서 330명 NPC 동시 운용
